## Requisitos do Projeto
### Requisitos Funcionais

|ID    | Descrição                | Prioridade                                                                                  |
|------|--------------------------|---------------------------------------------------------------------------------------------|
| RF-01 |A aplicação deve permitir ao usuário acessar informações sobre a área de desenvolvimento| ALTA | 
| RF-02 |A aplicação deve permitir ao usuário acessar lista de sub-área| ALTA |
| RF-03 |A aplicação deve permitir ao usuário acessar descrição detalhada da sub-área  selecionada | ALTA |
| RF-04 |A aplicação deve permitir ao usuário acessar planos de estudos para a sub-área selecionada| ALTA |
| RF-05 |A aplicação deve permitir ao usuário acessar melhores instituições validadas pelo MEC| ALTA |

**Prioridade: Alta / Média / Baixa. 

### Requisitos não Funcionais

|ID      | Descrição               |Prioridade                                                                                  |
|--------|-------------------------|--------------------------------------------------------------------------------------------|
| RNF-01 | A aplicação deve ser publicada em um ambiente acessível ao público na Internet| ALTA | 
| RNF-02 | A aplicação deverá ser responsiva permitindo a visualização aos diversos dispositivos de forma adequada |ALTA | 
| RNF-03 | A aplicação deve ser compatível com os principais navegadores: Google Chrome, Firefox e Microsoft Edge |ALTA | 

**Prioridade: Alta / Média / Baixa.
